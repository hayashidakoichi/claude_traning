## パート2：座学：MCPサーバー入門（30分）

### 2-1. MCPとは何か（7分）

> **MCP（Model Context Protocol）** とは、AIエージェントが外部サービスにアクセスするための共通規格です。Anthropicが提唱し、いまや業界標準になりつつあります。

たとえるなら **「AIエージェント業界の電源プラグ規格」** です。

```
┌────────────────────────────────────────────────────┐
│                                                    │
│   AIエージェント側     ⟷    MCPサーバー側          │
│   ・Claude Code               ・Google Drive       │
│   ・Cowork              ←→    ・Gmail              │
│   ・Claude.ai                 ・Google Calendar    │
│   ・他社のエージェント         ・社内システム       │
│                                                    │
│                                                    │
│   ↑どんなエージェントでも、同じプラグ規格でつながる ↑│
│                                                    │
└────────────────────────────────────────────────────┘
```

ポイント：
- 一度MCPに対応した外部サービスは、**Claude Code / Cowork / Claude.ai すべてから使えます**
- 他社のAIエージェント（OpenAI/Google等）も採用しており、**業界横断の標準**になっています
- 「自社の業務システムにMCPサーバーを立てる」というのが、AI時代のSaaSの新しい接続様式です

### 2-2. Skills と MCP の違い（10分）

ここがハマりやすいところです。**Skillsは「やり方」、MCPは「接続」** です。両方が揃って初めて業務が完結します。

| | Skills | MCP |
|---|---|---|
| 役割 | やり方を教える（手順・ノウハウ） | 外部サービスにつなぐ（接続・データ取得・アクション実行） |
| 中身 | Markdownの指示書＋テンプレ＋スクリプト | サーバープログラム（APIの窓口） |
| 提供形態 | フォルダ（ファイルの集まり） | 起動中のサーバー（ローカル/リモート） |
| 例 | 「議事録整形手順」「ニュースレター作成」 | 「Google Drive接続」「Gmail送信」「Calendar参照」 |
| たとえると | **レシピ本** | **食材を仕入れる業者** |

#### 具体例：週次レポート作成業務

```
【MCPの役割】= 食材を仕入れる
・Google Drive MCP → 元データのスプレッドシート取得
・Gmail MCP       → 関係者からの報告メール取得
・Calendar MCP    → 今週の主要会議取得

           ↓ 集めた素材

【Skillsの役割】= レシピに従って料理する
・週次レポートSkill → 決まった構成・トーンで組み立てる

           ↓ 仕上がり

【Gmail MCP】= 出来上がりを届ける
・自分宛にレポート完成通知メールを送る
```

> **TIPS**：Skillsだけだと「知識はあるが手が届かない」、MCPだけだと「手は届くが手順がバラバラ」になります。**両方揃って初めて業務が自動化できます。**

### 2-3. 3つのClaude製品でのSkills/MCP対応（10分）

「Claude」と一口に言っても、用途別に3つの入り口があります。それぞれSkills/MCPへの対応度が違います。

| | Claude Code（CLI） | Cowork（デスクトップ） | Claude.ai（Web/モバイル） |
|---|---|---|---|
| **操作方法** | ターミナル | デスクトップアプリ | ブラウザ・スマホアプリ |
| **見た目** | 黒い画面 | チャット画面 | チャット画面 |
| **Skills（利用）** | ◎ | ○ | △（組み込みのみ） |
| **Skills（自作）** | ◎ | ✕ | ✕ |
| **MCP（利用）** | ◎ | ○ | ○ |
| **MCP（自由に追加）** | ◎ | △（コネクタ経由のみ） | △（コネクタ経由のみ） |
| **ファイル操作** | ◎ | ◎（フォルダ単位） | ✕（アップロードのみ） |
| **学習コスト** | △ | ○ | ◎ |
| **向いている人** | 自動化を本格化したい人 | GUI派・日常業務メイン | チャットで相談したい人 |

> **本勉強会の方針**：3日目は Claude Code（CLI）でMCPを接続します。4日目で「同じMCPがCoworkでも動く」ことを確認し、日常業務にCoworkを取り込みます。

### 2-4. セキュリティの注意（3分）

- MCP接続時のOAuth：**何の権限を渡しているか必ず確認**してください
- **第三者の野良MCPサーバーは原則使いません**（公式・社内・信頼できるリポジトリのみ）
- 公式MCP一覧：[modelcontextprotocol.io](https://modelcontextprotocol.io)

## パート3：ハンズオン① Google MCP接続（35分）

### 3-1. 接続するMCP（一覧）

| MCP | 用途 |
|---|---|
| **Google Drive** | ファイル・フォルダの読み書き、Googleドキュメント/スライド/スプレッドシート操作 |
| **Gmail** | メール検索・送信・下書き作成 |
| **Google Calendar** | 予定の確認・作成・更新 |

> **補足**：「Google Chat への投稿もMCPでできないの？」とよく聞かれますが、**現時点でGoogle Chatの公式MCPサーバーは存在しません**。今日のクライマックスでは、自分への通知手段として **Gmail（自分宛送信／下書き作成）** を使います。

### 3-2. MCP追加の2つのやり方（7分）

MCPサーバーは **2つの場所** から追加できます。今回は **やり方B（Claude Codeを立ち上げた中で `/mcp` から追加）** を推奨します。画面が誘導してくれて、初めての方でも迷いにくいためです。

#### やり方A：ターミナルから `claude mcp add` で登録（上級者向け）

Claude Codeを起動する**前に**、ターミナルから直接登録するやり方です。

```bash
# 追加（書式）
claude mcp add <name> <command-or-url>

# 一覧確認
claude mcp list

# 削除
claude mcp remove <name>
```

例：Google Drive を追加する場合

```bash
claude mcp add --transport sse google-drive https://mcp.claude.ai/google-drive
```

#### やり方B：Claude Codeの中から `/mcp` で登録（推奨）

Claude Codeを立ち上げてから、対話画面の中で登録するやり方です。**OAuth画面まで画面が誘導してくれる**ので、こちらをおすすめします。今日は全員でこのやり方を体験します。

```bash
# まずClaude Codeを起動
claude
```

起動したら、入力欄に次のスラッシュコマンドを打ちます。

```
/mcp
```

→ MCP管理画面が開き、現在接続中のMCPサーバーとステータス（`connected` / `disconnected` / `needs auth`）が表示されます。ここから **「Add a server（サーバーを追加）」** を選んで新規追加します。

### 3-3. 3つのMCPを順番に接続（25分）

ここは講師が画面共有しながら **やり方B（Claude Code内の `/mcp` 画面）で1個ずつ全員で進めます**。OAuth画面では「何を許可しているか」を必ず読んでください。

#### 共通の手順（毎回これを繰り返します）

1. ターミナルで `claude` を実行して Claude Code を起動します（すでに起動済みならそのままでOK）
2. 入力欄に `/mcp` と打ってEnter → MCP管理画面が開きます
3. **「Add a server」** を矢印キーで選んでEnter
4. 名前（例：`google-drive`）を入力してEnter
5. 接続先のURLまたはコマンドを入力してEnter
6. ブラウザが自動で開き、Googleアカウントの **OAuth許可画面** が表示されます
7. **会社のGoogle Workspace アカウント** を選んで、「許可」を押します
8. ブラウザが「接続しました」と表示されたら、Claude Codeに戻ります
9. `/mcp` をもう一度開いて、ステータスが **`connected`** になっていることを確認します

#### Step 1：Google Drive MCP

- 名前：`google-drive`
- 接続先：講師から共有されたURL（例：`https://mcp.claude.ai/google-drive`）
- **OAuth画面で確認すること**：「Claudeに以下を許可する：ファイルの読み取り・編集・作成」
- 完了後、`/mcp` で `google-drive: connected` を確認

#### Step 2：Gmail MCP

- 名前：`gmail`
- 接続先：講師から共有されたURL
- **OAuth画面で確認すること**：「メール読み取り・送信・下書き作成」
- 完了後、`/mcp` で `gmail: connected` を確認

#### Step 3：Google Calendar MCP

- 名前：`google-calendar`
- 接続先：講師から共有されたURL
- **OAuth画面で確認すること**：「カレンダー閲覧・予定作成」
- 完了後、`/mcp` で `google-calendar: connected` を確認

> **TIPS**：1個追加するごとに `/mcp` で確認する癖をつけましょう。3個まとめて追加してから確認すると、どこで失敗したか分からなくなります。

### 3-4. 全接続の確認（3分）

最後にもう一度、Claude Code の中で次のコマンドを実行します。

```
/mcp
```

**ゴール**：3つすべて `connected` 状態になっていることを確認します。

```
google-drive     connected
gmail            connected
google-calendar  connected
```

> **トラブルシューティング**：接続できない場合は講師に声をかけてください。よくある原因は、(1)OAuthで個人アカウントを選んでしまった、(2)ブラウザのポップアップブロック、(3)会社のGoogle Workspace側で承認が必要、(4)`needs auth` のまま止まっている（→ `/mcp` 画面でそのサーバーを選んで「Authenticate」を再実行）、のいずれかです。

## パート4：ハンズオン② 自分宛Gmailに届ける ⭐（40分）

> **今日のクライマックス**：「Claudeに頼んだら、自分の受信トレイにまとめメールが届いた」を体験します。

### 4-1. 送信先の準備（3分）

- Gmail を開いてください
- 今日のテストでは **「自分のメールアドレスに自分から送る」** 形を取ります（自分の受信トレイに届きます）
- 受信トレイにテスト用ラベル（例：`Claudeテスト`）を作っておくと、後から見返しやすくなります（任意）

### 4-2. Step 1：単純な送信（7分）

最初の小さな成功体験です。

**プロンプト**：
```
自分のメールアドレス宛に、
件名「Claude Codeから初送信テスト」、
本文「Claude Codeから初めてメールを送りました」というGmailを送ってください。
```

**期待される動き**：
1. Claude が Gmail MCP を呼び出します
2. 「以下の内容で送信します。よろしいですか？」と確認されます
3. 承認すると送信完了です
4. 自分のGmail受信トレイにメールが届いていることを確認してください

> **ファシリTip**：ここでみんなが「届いた！」を声に出すと場が盛り上がります。

### 4-3. Step 2：カレンダー要約をメールで届ける（10分）

ここで「MCPの連携」を初体験します。

**プロンプト**：
```
今日のGoogle Calendarの予定を要約して、
自分宛のGmailで届けてください。

件名は「【本日の予定サマリー】」で始めてください。
本文は時系列で、時間・予定名・参加者がわかるようにまとめてください。
```

**期待される動き**：
1. Calendar MCP で今日の予定を取得します
2. Claudeが内容を整形します
3. Gmail MCP で自分宛にメールを送信します

> **「データを取る → 整形する → 届ける」という連携の感覚**を掴んでください。これがあとで業務自動化の発想の起点になります。

### 4-4. Step 3：未読メールの仕分け → Gmail下書きで残す（10分）

Step 3 では Gmail MCP のもう一つの便利機能 **「下書き作成」** を使います。**勝手に送らずに、人が見てから送る** という業務に向いています。

**プロンプト**：
```
未読のGmailのうち、自分宛で返信が必要そうなものを3件選んで、
それぞれ「送信者／件名／要点1行／推奨する次のアクション」をまとめてください。

そのまとめを本文にして、件名「【未読メール仕分け】返信候補3件」のGmail下書きを
自分宛で1通だけ作成してください。送信はせず、下書きフォルダに残してください。
```

**期待される動き**：
1. Gmail MCP で未読メールを取得します
2. Claudeが「返信が必要そうなもの」を判断します
3. Gmail MCP で **下書き（Draft）** を1通作成します（送信はしません）
4. Gmail の「下書き」フォルダを開いて、内容を確認してください

> **TIPS**：「重要度の判断」を AI に任せられるかどうかは、人によって信頼度が分かれます。下書きにしておけば、自分の目で確認してから送信できるので、最初の導入として安全です。

### 4-5. Step 4：scheduleコマンドで1分後に未読メール要約を自分宛Gmailへ（10分）

ここまでは「頼んだら、その場ですぐ動く」体験でした。最後に **「いま頼んで、あとで自動的に動かす」** を体験します。Claude Code には `/schedule` コマンドがあり、**1回だけの予約実行や定期実行**を仕掛けられます。

お題は Step 3 に似た「未読メールの要約を自分宛に届ける」ですが、今回はそれを **1分後に自動実行** させ、しかも **下書きではなく自分宛に送信** します。**同じような作業を「あとで」やってもらう** ことで、`/schedule` のうれしさが分かります。

> **業務イメージ**：本来は **「毎朝9時に今日のメールサマリーを自分宛Gmailで送る」** のように使います。今日は勉強会の時間内に結果を見たいので、お試しとして1分後で動かします。同じ要領で「会議の10分前にリマインドメール」「夜のうちに翌日の予定をメールで整理しておく」なども仕掛けられます。

**プロンプト**：
```
/schedule 1分後に1回だけ、以下を実行してください。

1. 未読のGmailのうち、自分宛で返信が必要そうなものを3件選んでください
2. それぞれ1行で要約してください（送信者・件名・要点）
3. まとめを本文にして、件名「【予約配信】未読メールサマリー」のGmailを
   自分のメールアドレス宛に送信してください

本番運用では「毎朝9時に」と指定する想定ですが、今日は動作確認のため1分後にします。
```

**期待される動き**：
1. `/schedule` が「1分後に1回だけ実行」のタスクとして予約されます
2. 約1分後、裏側でClaudeが起動して、Gmail MCP で未読メールを取得 → 要約します
3. Gmail MCP で自分宛にサマリーメールが送信されます
4. 自分のGmail受信トレイで「【予約配信】未読メールサマリー」が届いたか確認してください

**チェックポイント**：
- 受信トレイに「【予約配信】未読メールサマリー」が届いている
- 3件分の要約（送信者・件名・要点）が本文で読める
- Step 3 と似た結果が、**自分の手を動かさずに**届いた感覚を味わえる

> **ファシリTip**：1分の待ち時間は短い雑談タイムにあてましょう。「これを毎朝9時に動かしたら、出社直後の30分が浮きませんか？」と問いかけると、5-3のディスカッションへの良い助走になります。

> **トラブルシューティング**：
> - 1分経っても届かない場合は、`/schedule` の一覧コマンドで予約状況を確認してください
> - 「未読メールが0件です」と返ってきた場合は、お題を「最新の受信メール3件」に変えて再度試してください
> - 権限エラーが出る場合は、Gmail MCP が `connected` になっているかを `/mcp` で確認してください

## パート5：ハンズオン③ MCP連携の応用（35分）

> ここからは **「自分の業務に置き換えたら？」** を考えながら進めます。

### 5-1. Driveのファイルを読んで要約 → 自分宛Gmailに送る（12分）

**プロンプト例**：
```
Google Driveの「会議資料」フォルダにある最新のドキュメントを読んで、
3行で要約して、自分宛のGmailで届けてください。
件名は「【資料サマリー】<元ファイル名>」にしてください。
```

**ポイント**：
- 「最新の」のような曖昧な指示でもMCPは動きます
- ファイルを開かなくても、Claude が中身を直接読めます

### 5-2. 週次の予定を整形 → 自分宛Gmailにまとめる（10分）

**プロンプト例**：
```
今週月曜から金曜までのGoogle Calendarの予定を
日付・時間・予定名・参加者の表形式に整理して、
件名「【週次予定まとめ】」のGmailで自分宛に送ってください。
```

**ポイント**：
- カレンダー画面で見るより、表で渡されたほうが俯瞰しやすいです
- 月初・週初の「振り返り＆計画」に使えます

### 5-3. ディスカッション：「自分の業務だったら何ができる？」（13分）

**ワーク**：3〜4人グループになって、以下を出し合ってみましょう。

| 自分の業務 | 使えそうなMCPの組み合わせ | 期待される効果 |
|---|---|---|
| 例：毎朝のメールチェック | Gmail → Gmail（下書き） | 重要メールだけ仕分けて、返信ドラフトを下書きに残す |

| 例：顧客対応ログ | Drive + Gmail → Drive | 顧客ごとのやり取りを1ファイルにまとめる |

最後に3〜4人が代表してアイデアを発表してください。

## 番外編：Linear MCP の追加（チケット管理を Claude から触る）

> **対象**：「3つの Google MCP は接続できた、もう少しMCPを増やしてみたい」という方向け。Linearを業務で使っている方も、これから検討する方も、両方に役立ちます。

### この番外編で得られるもの

- **Linear**（プロジェクト・チケット管理ツール）の MCP を Claude Code につなぐ手順を体験できます
- Google Workspace 以外の SaaS も **同じ「OAuth → `/mcp`」** の流れでつながることを確認できます
- 「Linearからチケット一覧を取得して、要点を自分宛Gmailにまとめる」など、**本日の連携パターンが他のSaaSでもそのまま応用できる**感覚を掴めます

### Linear とは

Linearは、ソフトウェアチーム向けのプロジェクト・課題管理ツールです（Jira / Notion / Asana の仲間です）。**公式の MCPサーバーを提供している**ため、Claude Code からチケットの読み書きが可能です。

| 項目 | 内容 |
| --- | --- |
| サービスURL | <https://linear.app> |
| 公式MCP URL | `https://mcp.linear.app/sse` |
| 認証方式 | OAuth（ブラウザで Linear ワークスペースへの許可） |
| 無料プラン | あり（個人利用・小規模チーム向けで十分試せます） |

### Step 1：Linear アカウントの用意（5分）

1. <https://linear.app> にアクセスします
2. **「Sign up」** から Google または メールアドレスでサインアップします
3. ワークスペース名を入力します（会社で使う場合は会社名、個人なら自分の名前でOK）
4. 最初のチームを作成します（例：`Personal` や `テスト`）
5. **動作確認用にダミーのチケットを2〜3件作っておきます**（タイトル・担当者・期限を入れたものを用意）

> **すでに業務でLinearを使っている方**：そのままご自身のワークスペースで進めてOKです。ただし、他メンバーに見えるチケットを触る可能性があるので、最初の確認は **自分にアサインされているチケットだけを対象** に行うのが安全です。

### Step 2：Claude Code から Linear MCP を追加（5分）

パート3-2で学んだ **やり方B（Claude Code内の `/mcp` から追加）** をそのまま使います。

1. ターミナルで `claude` を実行して Claude Code を起動します（すでに起動していればそのまま）
2. 入力欄に `/mcp` と打ってEnter → MCP管理画面が開きます
3. **「Add a server」** を矢印キーで選んでEnter
4. 名前：`linear` と入力してEnter
5. 接続先：`https://mcp.linear.app/sse` と入力してEnter
6. ブラウザが自動で開き、**Linear の OAuth許可画面** が表示されます
7. ワークスペースを選び、**「Authorize Claude」** をクリックします（読み書き権限の付与）
8. ブラウザに「接続しました」と表示されたら、Claude Code に戻ります
9. `/mcp` をもう一度開き、ステータスが **`linear: connected`** になっていることを確認します

> **ターミナルから一発で追加したい場合**（やり方A）は次のコマンドでもOKです：
> ```bash
> claude mcp add --transport sse linear https://mcp.linear.app/sse
> ```
> その後 `claude` を起動して `/mcp` から認証を完了させてください。

### Step 3：動作確認 — チケット一覧を取得してみる（3分）

最初の動作確認プロンプトです。

**プロンプト**：
```
Linearで自分にアサインされているチケットを5件、
タイトル・状態（Todo / In Progress / Done など）・期限の一覧で見せてください。
```

期待される動き：
1. Linear MCP が呼び出されます
2. 自分にアサインされているチケットを取得します
3. 一覧として返ってきます

ここまで返ってくれば接続成功です。

### Step 4：応用 — 本日の連携パターンを Linear に当てはめる（任意）

Google MCP のときと同じ **「データ取得 → 整形 → 配信」** を、Linear でも試してみましょう。

**プロンプト例**：
```
Linearで今週中（金曜まで）に締切が来る自分のチケットを一覧化して、
件名「【今週の締切チケット】」のGmailで自分宛に送ってください。

本文には、各チケットについて
- タイトル
- 状態
- 締切日
- Linear上のURL
を含めてください。
```

> **発想のヒント**：「Slack MCPだったら？」「Notion MCPだったら？」と置き換えて考えてみると、**MCPは増やせば増やすほど業務全体が連動する** という感覚が持ち帰れます。

### よくあるつまずき

- `/mcp` で `needs auth` のまま動かない場合 → ワークスペース選択でミスがある可能性があります。一度 `claude mcp remove linear` で削除して再追加してください
- 「権限がない」エラーが出る場合 → Linear 側で **メンバー権限以上** が必要です。ワークスペース管理者にロール確認を依頼してください
- **Linearを業務で使っていない方** → 個人ワークスペース（Personal）を作り、家事タスクや学習計画など **個人用の管理** から触り始めるのがおすすめです。MCP連携の感覚は同じです
