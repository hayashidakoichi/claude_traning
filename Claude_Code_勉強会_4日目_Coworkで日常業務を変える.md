## 4日目のストーリー

4日目で MCP が繋がった。Claude が Google Workspace を読み書きできるようになった。

でも正直、**日常業務を全部ターミナルでやるのは疲れる**。メールの確認をするのに `claude` コマンドから始めるのは、ちょっと大げさ。

実は **同じ MCP・同じ Skill が、ターミナル不要の Cowork でも動く**。

今日は Cowork に乗り換えて、メール・資料といった**日常業務を高速化**する。そして最後はクライマックス：

**「Markdown原稿 → 人間レビュー → マッキンゼー風Marp → Googleスライド」**

を一気通貫でやり切る。**経営層に提出できる品質のスライドが、Claudeから出てくる**ところまで。

> **今日のゴール**：マッキンゼー風のGoogleスライドを、Claudeに作らせて「実際に使える」品質にする。

## パート2：座学：Code と Cowork の違い（25分）

### 2-1. なぜ Cowork という選択肢があるのか（5分）

> Anthropic が Cowork を作った理由：「**Claude Code は強力だが、ターミナルに慣れていない人には使いづらかった**。同じエージェント技術を、デスクトップアプリの形で誰でも使えるようにしたい」

```
[ Claude のエージェント技術 ]
              │
              ├─────────────┐
              ▼             ▼
       Claude Code         Cowork
       （CLI）            （デスクトップ）
       
       中身は同じ。インターフェースだけ違う。
       
       同じMCP・同じSkillが両方で動く ←★ ここが重要
```

つまり Cowork は「CLI が苦手な人のための代替手段」ではなく、**「日常業務向けに最適化された、もう一つの出口」**。

### 2-2. Claude Code でできること／苦手なこと（5分）

#### Claude Code（CLI）でできること

| カテゴリ | できること | 具体例 |
|---|---|---|
| ファイル操作 | ✅ ローカル全ファイルへ自由にアクセス | あらゆるフォルダの読み書き |
| コマンド実行 | ✅ あらゆるターミナルコマンド | Python実行・Git操作・パッケージインストール |
| Git管理 | ✅ コミット・ブランチ・PR作成まで自動 | 変更履歴を残す・やり直し可能 |
| Skills | ✅ 自作・配布・スコープ管理 | `.claude/skills/`配下で完全自作 |
| MCP | ✅ ローカル/リモート両方追加可 | `claude mcp add`で自由に拡張 |
| 並列実行 | ✅ 複数セッションを同時に走らせられる | 1つ目で資料作成、2つ目で集計 |
| スケジュール起動 | ✅ HooksやGitHub Actionsと連携 | 定期実行・トリガー実行 |
| 速度 | ✅ UIを経由しないので最速 | レスポンスが体感で速い |

#### Claude Code（CLI）で苦手なこと

| ❌ 苦手 | 理由 |
|---|---|
| GUI操作 | コマンドベースなので「目で見ながら」の操作はしづらい |
| 進捗の視覚的確認 | 黒い画面でテキストが流れるだけなので圧倒される人もいる |
| 非エンジニアへの教育 | ターミナル操作・コマンドの概念に慣れる必要がある |
| 日常の軽い作業 | わざわざターミナルを開くのが面倒に感じる |

### 2-3. Coworkでできること／できないこと（10分）

ここが**最重要**。CLIから来た人がよく引っかかる。

#### Coworkでできること

![Cowork が日常業務を自動化](https://cdn.prod.website-files.com/6889473510b50328dbb70ae6/6972a1d3d9f82138ceba7cb9_cowork-how-01.webp)
*出典: [Anthropic — Cowork product page](https://claude.com/product/cowork)*

| カテゴリ | できること | 具体例 |
|---|---|---|
| ファイル操作 | ✅ 指定したフォルダ配下を読み書き | フォルダごとに権限を細かく付与 |
| Skills（利用） | ✅ 配布されたSkillがそのまま動く | 同僚が作ったSkillをそのまま利用 |
| MCP | ✅ コネクタとして接続可 | Google Drive/Gmail/Calendar/Chatなど |
| ファイルD&D | ✅ 画面にドラッグ&ドロップで投入 | PDF/Excelなどまとめて投入 |
| スケジュールタスク | ✅ 毎朝/毎週など自動実行 | 毎朝のメール要約・週次レポート |
| モバイルから指示 | ✅ スマホアプリから指示送信 | 通勤中に依頼、デスクトップで実行 |
| コンピューター操作 | ✅ アプリ起動・ブラウザ自動操作 | ブラウザから情報収集・転記 |
| プロジェクト機能 | ✅ 関連タスクを1つの作業空間にまとめられる | 案件ごとにファイル・指示・記憶を保持 |
| サンドボックス実行 | ✅ 仮想マシン内で安全に動作 | 指定外フォルダには触れない設計 |

#### Coworkでできないこと（要注意）

| ❌ できない | 理由・回避策 |
|---|---|
| **Skillsの自作** | Cowork単体では作れない → **Claude Codeで作って配布する** |
| **Git連携** | バージョン管理機能がない → 元ファイルのバックアップは手動 |
| **並列セッション** | 1つのタスクが終わるまで次は走れない |
| **ローカルMCP（カスタム）** | 自分のPCで動くカスタムMCPサーバーの追加は不可。公式コネクタのみ |
| **アプリを閉じると停止** | **デスクトップアプリを開いておく必要がある**。閉じるとタスク停止 |
| **規制業務での利用** | HIPAA・FedRAMP・金融規制業務では使用不可 |
| **速度** | UI経由のオーバーヘッドあり。CLIより遅い |

### 2-4. 早見表：Code vs Cowork vs Claude.ai（5分）

| | Claude Code（CLI） | Cowork（デスクトップ） | Claude.ai（Web/モバイル） |
|---|---|---|---|
| **操作方法** | ターミナル | デスクトップアプリ | ブラウザ・スマホアプリ |
| **見た目** | 黒い画面 | チャット画面 | チャット画面 |
| **ファイル操作** | ◎ ローカル自在 | ◎ フォルダ単位で許可 | ✕ アップロードのみ |
| **Skills自作** | ◎ | ✕ | ✕ |
| **Skills利用** | ◎ | ○ | △ |
| **MCP（追加）** | ◎ | △ | △ |
| **MCP（利用）** | ◎ | ○ | ○ |
| **コマンド実行** | ◎ | △ | ✕ |
| **Git連携** | ◎ | ✕ | ✕ |
| **並列実行** | ◎ | △ | △ |
| **速度** | ◎ | ○ | ◎ |
| **学習コスト** | △ | ○ | ◎ |
| **アプリ常時起動** | 不要 | **必要（閉じると停止）** | 不要 |
| **向いている用途** | Skill開発・自動化 | 日常業務・資料作成 | 調査・相談・壁打ち |

> **覚えてほしい一言**：**「作るのはCode、使うのはCowork」**
> Codeで作ったSkillが、Coworkでもそのまま動く。これが分業の鍵。

## パート3：Coworkセットアップ＆MCP接続（20分）

### 3-1. インストール確認（5分）

#### Windows版のインストール要件

| 項目 | 内容 |
|---|---|
| OS | Windows 10/11（x64） |
| 必須機能 | Virtual Machine Platform（仮想マシンプラットフォーム） |
| インストーラ | MSIX形式（管理者権限必要） |
| ストレージ | VMイメージ約2GBを初回ダウンロード |
| プラン | Pro / Max / Team / Enterprise いずれか |

ダウンロード：<https://claude.com/download>

> ⚠️ **管理者権限が必須**：CoworkはWindows上で仮想マシンを使って動くので、インストールに管理者権限が必要。会社PCに制限がある場合は事前に情シスへ確認。

### 3-2. 初回起動とフォルダ許可（10分）

```
[ 起動の流れ ]

1. Claude Desktopアプリを起動
        ↓
2. 上部メニューで "Cowork" タブを選択
        ↓
3. "Work in a Folder" にチェック
        ↓
4. 作業用フォルダを選択（例: ~/Documents/claude-cowork-work/）
        ↓
5. アクセス許可ダイアログで "Allow" を選択
```

> **TIPS**：作業用フォルダは**機密情報を含まない場所**にする。最初はデスクトップに `cowork-work` フォルダを作るのが楽。

### 3-3. Google MCPの接続（5分）⭐

**ここが今日の感動ポイントの一つ**。

4日目で Claude Code に接続した **同じ Google MCP** が、Cowork でもそのまま使える。

#### 接続手順
1. Cowork 画面の左サイドバーから "Connectors" を選択
2. Google Drive / Gmail / Calendar / Chat を選んで接続
3. OAuth は4日目と同じアカウントを使う（多くの場合、再認証不要）

#### 確認のプロンプト
```
今日のGoogle Calendarの予定を教えてください。
```

→ 4日目と全く同じように動く。

> **TIPS**：「同じMCPが両方で使える」を体感するのが、このパートのゴール。「**自分はCLI、同僚はCowork**」という分業が現実的になる瞬間。

## パート4：ハンズオン① メール業務をCoworkで（25分）

> ターミナルを開かずに、Coworkのチャット画面だけで業務をやってみる。

### 4-1. 課題1：今朝のメール要約（8分）

**プロンプト**：
```
Gmailの今朝届いた未読メールのうち、
自分宛のものを5件選んで、それぞれ3行で要約してください。
返信が必要なものには「要返信」マークをつけて。
```

**期待される動き**：
- Gmail MCP で未読メールを取得
- 5件選定 → 要約 → 「要返信」判定
- チャット画面に整形されたリスト表示

### 4-2. 課題2：返信ドラフトを書く（10分）

**プロンプト**：
```
さっきの5件のうち、要返信のものに対して
丁寧な返信ドラフトを3パターン（簡潔／標準／丁寧）作ってください。
```

**ポイント**：
- Coworkの「複数パターン同時生成」が活きる
- 自分で選んで→Gmail下書きに反映、までを一連でやれる

### 4-3. 課題3：週次のメール棚卸し（7分）

**プロンプト**：
```
今週月曜からのGmailで、
自分が出したメールと受け取ったメールを集計し、
やり取り件数の多い相手トップ5と、主なトピックを整理してください。
```

**ポイント**：
- 普段やらない人にとっては「**えっ、こんなことできるの**」という驚き
- 週次の振り返り・1on1の素材として使える

## パート5：ハンズオン② マッキンゼー風スライド生成 ⭐⭐（60分）

> **今日の大クライマックス**：経営層に出せるGoogleスライドを、Coworkだけで完成させる。

### 5-1. 全体フロー（5分）

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│   ステップ1: Markdownで原稿生成                              │
│      Coworkに「○○の提案書をMarkdownで作って」                │
│                  ↓ output.md                              │
│                                                          │
│   ステップ2: 人間がレビュー・修正                              │
│      内容・固有名詞・数字を確認                                │
│                  ↓ 編集済みoutput.md                       │
│                                                          │
│   ステップ3: マッキンゼー風Marpに変換                          │
│      Coworkに「これをマッキンゼー形式のMarpに変換して」          │
│                  ↓ slides.md                             │
│                                                          │
│   ステップ4: Marpプレビューで確認・微調整                      │
│                  ↓ 仕上がりOK                              │
│                                                          │
│   ステップ5: GoogleスライドへアップロードしてShow & Tell        │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 5-2. 事前用意：マッキンゼー風MarpテンプレSkill（講師事前作成済）

```
.claude/skills/mckinsey-slides/
├── SKILL.md
├── references/
│   ├── slide-principles.md   # 1スライド1メッセージ、ピラミッド原則
│   ├── layout-patterns.md    # タイトル/キーメッセージ/サポート構造
│   └── marp-template.md      # Marpテーマ定義
└── examples/
    └── sample-deck.md
```

このSkillは **Claude Code（CLI）で作成済**で、配布されている。Coworkに同じフォルダを置けば、そのまま動く。

> **メッセージ性**：「Claude Code で作った Skill が、Cowork でそのまま使える」を実感するパート。

### 5-3. お題（5分）

各自、以下から1つ選ぶ（または自由テーマ）：
- 自分の部門の今期の取り組み（経営層向け3枚）
- 新規顧客への提案概要（営業向け5枚）
- 社内ツール導入の提案（IT部門向け3枚）

### 5-4. ステップ1：Markdownで原稿生成（10分）

**プロンプト例**：
```
以下のテーマで提案書をMarkdownで作成してください。
出力先：output.md

【テーマ】
○○部門の2026年Q2の重点取り組み

【構成】
- エグゼクティブサマリー
- 現状の課題
- 提案する取り組み（3つ）
- 期待される効果
- 実行スケジュール
```

→ `output.md` が作業フォルダに生成される。

### 5-5. ステップ2：人間がレビュー・修正（15分）

**ここが一番大事なパート**。

- Markdownを開いて中身を読む
- AIが書いた内容で **「ここはAIに任せず人間が決めるところ」を意識する**
- 例：固有名詞・数字・社内事情・トーン・現実とのズレ

> **TIPS**：AIに「全部任せる」のではなく、「AIに**叩き台**を作らせて人間が**監修する**」というスタンスが、現時点では一番質が高くなる。

### 5-6. ステップ3：マッキンゼー風Marpに変換（10分）

**プロンプト**：
```
output.md を、mckinsey-slides Skill を使って
マッキンゼー形式のMarpに変換してください。
出力先：slides.md
```

→ `slides.md` がMarp形式で生成される。

#### マッキンゼー形式のポイント
- **1スライド1メッセージ**：キーメッセージは1つだけ
- **ピラミッド原則**：結論を先に、根拠を後に
- **3点構成**：サポート要素は3つに絞る（多すぎない・少なすぎない）

### 5-7. ステップ4：Marpプレビューで確認・微調整（10分）

#### VS Code Marp拡張でのプレビュー
1. VS Code を開く
2. `slides.md` を表示
3. 右上の "Open Preview" でMarpプレビューを表示
4. 微調整は再度Coworkに依頼するか、自分でMarkdown編集

または、CoworkにMarp対応プレビューが入っていればそちらで。

### 5-8. ステップ5：GoogleスライドへアップロードしてShow & Tell（5分）

**プロンプト**：
```
slides.md をPPTX形式で書き出して、
Google Driveの「2026Q2-提案」フォルダに保存してください。
Googleスライドとしてインポートできる状態にしてください。
```

→ Google Drive にアップロード完了

最後に**全員のスライドを並べて Show & Tell**。「これがClaudeから出てきた」を共有する瞬間。

> **講師Tip**：このパートが感動の山場。「**経営層に持っていけるレベル**」かどうかを口頭で確認しながら、品質感を全員でキャリブレーションする。
