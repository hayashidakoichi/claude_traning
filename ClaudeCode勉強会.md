### 勉強会の目的

Claude はプロの開発者が最も支持しているツールだが、決して難しいツールではない。自然言語（日本語）で指示を出すだけで AI が作業を遂行するため、**コーディング経験は不要**。

プロが使う最先端のツールを、ぜひ皆さんにも使いこなしてほしい — それが本勉強会の目的である。

## Claude Code はどこまでできるのか？ どこまで業務効率化を目指すのか？

税理士が一人でアシスタントゼロ、クライアント 60 社を回している事例がある。Claude Code で記帳・書類作成・顧客対応などの業務をなるべく自動化し、通常なら数名のスタッフが必要な業務量を一人でこなしている。

これは現時点での Claude Code による業務効率化の最高峰の例であり、本勉強会が目指すゴールの先にある姿でもある。

> 詳細は [AI税理士.pdf](refs/AI税理士.pdf) を参照。

## 目指すべき姿 — 専門分野を持つ私たちにできること

上記の税理士の事例が「最高峰」だとすれば、各人が持つ専門分野を持つ私たちの場合はどうか。たとえば以下のような業務が、Claude Code で大幅に効率化できる。

| 業務 | 従来の進め方 | Claude Code を活用すると |
|---|---|---|
| 競合調査 | 各社サイトを1つずつ確認し、手作業で比較表を作成 | 3社のURLを渡すだけで比較表が完成 |
| 提案書作成 | テンプレートを探し、顧客に合わせて手作業で編集 | 業種・規模・課題を伝えるだけでカスタム提案書を生成 |
| セミナー準備 | 案内メール・LP・資料を個別に作成 | 一連の成果物をまとめて指示し、一気に生成 |
| 評価データ集計 | Excel で関数を組み、グラフを作成 | CSV を渡して「集計してレポートにして」で完了 |
| 個別フィードバック | 30人分のレターを1通ずつ文面を変えて作成 | 評価データから30人分を一括生成、評価に応じてトーンも自動調整 |

**一人ひとりが Claude Code を使いこなすことで、今の 10倍の業務をこなせるようになる** — この勉強会は、その第一歩である。

## なぜ今 Claude（特に Claude Code）なのか

![Claude Code](https://cdn.prod.website-files.com/6889473510b50328dbb70ae6/68c469d41149ace562bfd24d_og-claude-product-claude-code.jpg)
*出典: [Anthropic — Claude Code 製品ページ](https://claude.com/product/claude-code)*

AIアシスタントは数多く存在するが、なぜ今 Claude を学ぶのか。まずは主要 3 サービスの位置づけを俯瞰し、Claude が注目されている理由を確認する。

### 3 大 AI サービス全体像

| | ChatGPT | Gemini | Claude |
|---|---|---|---|
| 提供会社 | OpenAI | Google | Anthropic |
| 代表モデル | GPT-4o / o3 | Gemini 2.5 Pro / Flash | Claude Opus 4 / Sonnet 4 |
| Web チャット | ChatGPT（chat.openai.com） | Gemini（gemini.google.com） | Claude（claude.ai） |
| デスクトップアプリ | ChatGPT Desktop | Antigravity | Claude Desktop |
| AI エージェント | Operator / Codex | Project Mariner（実験中） | Claude Code |
| CLI ツール | Codex CLI | Gemini CLI | **Claude Code** |

> **CLI（Command Line Interface）ツール**とは、ターミナル（コマンドライン）から AI を操作するツールのこと。3 社とも CLI を提供しているが、本勉強会で学ぶ **Claude Code** が最も早く実用化され、開発者からの支持も最も高い。

### 3 大 AI アシスタント比較

| | ChatGPT（OpenAI） | Gemini（Google） | Claude（Anthropic） |
|---|---|---|---|
| 得意分野 | 幅広い汎用タスク・プラグイン連携 | Google サービスとの統合・検索連動 | 長文理解・正確な推論・コーディング |
| ユーザー規模 | 週間アクティブ 4 億人超 | Google 製品に統合し急拡大中 | デイリーアクティブ 1,130 万人（急成長中） |
| コンテキストウィンドウ | 128K トークン（PDF 約 300 ページ分） | 1M トークン（PDF 約 1,500 ページ分） | 1M トークン（PDF 約 2,500 ページ分） |
| 特徴 | 最も早く普及した先駆者 | Google 検索・Workspace と一体化 | 安全性と正確性を重視した設計 |

**コンテキストウィンドウ**とは、AI が 1 回の会話で読み込める情報量の上限のこと。この数字が大きいほど、長い文書や大量のファイルをまとめて渡しても内容を把握できる。Claude と Gemini はともに 1M トークンだが、同じトークン数でも Claude の方が多くのページを処理できる。さらに Claude は読み込んだ内容の正確な理解と推論の質で高い評価を得ている。

ChatGPT は先行者として圧倒的なユーザー数を持ち、Gemini は Google エコシステムとの統合が強み。一方 Claude は**正確性・安全性・コーディング能力**で急速に支持を拡大している。

> **TIPS** — ChatGPT や Gemini が「使えない」わけではない。用途によって最適な AI は異なる。ただし「正確さ」と「コーディング能力」が求められる業務自動化の文脈では、Claude が現時点で最も高い評価を得ている。普段の調べ物には ChatGPT、業務の自動化には Claude、というように使い分けるのも一つの手。

### Claude が今最も注目されている 3 つの理由

**理由 1：アプリダウンロード数で ChatGPT を抜いて 1 位に**

2026 年 3 月、米国の App Store・Google Play で Claude が ChatGPT を抜いてダウンロード数 1 位を獲得した。1 日あたりのダウンロード数は Claude 14.9 万件に対し ChatGPT は 12.4 万件。1 日 100 万人以上が新規登録し、デイリーアクティブユーザーは 1,130 万人に達している。

**理由 2：売上成長率で OpenAI を急追**

Anthropic の年間売上ランレート（ARR）は 190 億ドルに到達し、OpenAI の 250 億ドルに迫っている。成長率は年 10 倍ペースで、2026 年中盤にも逆転する見通しが報じられている。

**理由 3：米軍の作戦計画にも採用される信頼性**

Claude は米軍が作戦の情報分析に採用するほど、AI としての信頼性が高く評価されている。安全性を最優先に設計された Anthropic の思想が、最もシビアな現場でも認められている。

> **TIPS** — AI 業界の数字は月単位で変わる。ここで挙げた数値は 2026 年 3 月時点のもの。最新の状況が気になったら「Claude 最新ニュース」で検索してみよう。

### 開発者が選ぶ AI コーディングツール No.1

Claude Code はリリースからわずか 8 ヶ月で、最も使われている AI コーディングツールの地位を確立した。

- **SWE-bench**（ソフトウェアエンジニアリングベンチマーク）で **74.4%** のスコアを記録し、全 AI ツール中トップ
- 開発者の「最も好きなツール」調査：**Claude Code 46%** vs Cursor 19% vs GitHub Copilot 9%

> **TIPS** — 「プロのエンジニアが最も使いたいツール」と聞くと難しそうに感じるかもしれないが、操作は日本語で指示を出すだけ。エンジニアと同じ道具を、エンジニアでなくても使えるのが Claude Code の魅力。

### Anthropic という会社 — なぜ開発者に信頼されるのか

![Dario Amodei（Anthropic CEO）](https://upload.wikimedia.org/wikipedia/commons/thumb/e/e4/Dario_Amodei_at_TechCrunch_Disrupt_2023_01_%28cropped%29.jpg/960px-Dario_Amodei_at_TechCrunch_Disrupt_2023_01_%28cropped%29.jpg)
*Dario Amodei（Anthropic CEO） — Photo: TechCrunch, CC BY 2.0, via [Wikimedia Commons](https://commons.wikimedia.org/wiki/File:Dario_Amodei_at_TechCrunch_Disrupt_2023_01_(cropped).jpg)*

Anthropic は OpenAI の元副社長 Dario Amodei と、同じく OpenAI で安全性研究を率いた Daniela Amodei（姉弟）が 2021 年に設立した会社である。「AI の安全性を最優先にする企業が必要だ」という信念から、OpenAI を離れて創業した。

この出自が、Anthropic の製品思想に深く影響している。

- **Constitutional AI（憲法 AI）** — Claude には「AI がどう振る舞うべきか」の原則が組み込まれている。有害な出力を減らしつつ、有用性を犠牲にしない設計思想
- **安全性へのコミットメント** — 能力を上げることと安全性を高めることを対立させず、両立させるアプローチを貫いている
- **透明性** — モデルの限界や弱点を隠さず公開し、安全性に関する研究論文を積極的に発表している

こうした姿勢が「信頼できる AI を使いたい」と考える開発者・企業から強く支持されている。

### AI 規制をめぐる Anthropic の姿勢

AI 業界では規制に反対する企業が多い中、Anthropic は一貫して**適切な規制は必要だ**という立場を取っている。

2025 年、カリフォルニア州で AI 規制法案（SB 1047）が議論された際、OpenAI や Meta など大手テック企業が反対に回る中、Anthropic は修正案を支持する書簡を公開した。「AI が社会にリスクをもたらす可能性がある以上、業界が自主的に安全基準を設けるべきだ」という主張である。

2026 年に入ると、トランプ政権が AI 規制の撤廃・緩和を進める中でも、Anthropic は自社の安全基準を下げない方針を明確にしている。政府が規制を緩めても、企業として守るべきラインは守る — この姿勢が、セキュリティを重視する企業や政府機関からの信頼につながっている。

> **TIPS** — 社内の機密データや顧客情報を扱う業務に AI を使うなら、安全性を最優先にしている会社のツールを選ぶのが合理的。Anthropic のこうした姿勢は、自社で AI を導入する際の選定理由としても説得力がある。

### Claude Code のセキュリティとデータプライバシー

![Claude Code Security](https://cdn.prod.website-files.com/68a44d4040f98a4adf2207b6/69050f2159f55e8c6f5f8e0f_og-claude-generic.jpg)
*出典: [Anthropic](https://www.anthropic.com/news/claude-code-security)*

業務で AI を使うとき、最も気になるのは「入力した情報が AI の学習に使われないか」「社内データが外部に漏れないか」という点だろう。Claude Code はこの懸念に対して、他の AI ツールより明確な保証を提供している。

**入力データは AI の学習に一切使われない**

Claude Code で入力したプロンプト、読み込ませたファイル、PC 内のデータは、Claude の AI モデルのトレーニング（学習）に使用されない。これは Anthropic の利用規約で明確に保証されている。他の AI ツールではオプトアウト（利用者が自分で設定を変えて学習対象から外す）が必要な場合があるが、Claude はデフォルトで学習に使わない。

| | Claude（Anthropic） | ChatGPT（OpenAI） | Gemini（Google） |
|---|---|---|---|
| 入力データの学習利用 | **されない**（デフォルト） | 設定でオプトアウトが必要 | 無料版は学習に使用、有料版は除外 |
| ファイル内容の学習利用 | **されない** | API 版は除外、Web 版は要設定 | 無料版は学習に使用、有料版は除外 |

**データの処理と保存**

- 会話データは処理のために一時的にサーバーに送信されるが、モデルのトレーニングには使用されない
- Team / Enterprise プランでは、会話ログの保持期間やアクセス制御を管理者が設定できる
- Claude Code はローカル（自分の PC）で動作し、ファイルの読み書きも PC 上で完結する。クラウドにファイルがアップロードされるわけではない

**3 大 AI のセキュリティ比較**

| セキュリティ項目 | Claude（Anthropic） | ChatGPT（OpenAI） | Gemini（Google） |
|---|---|---|---|
| データの学習利用 | **しない**（デフォルト） | オプトアウトが必要 | 無料版は利用、有料版は除外 |
| ローカル実行 | **PC 上で動作**（Claude Code） | クラウド処理 | クラウド処理 |
| 権限制御 | ファイル・コマンド単位で許可/拒否を設定可能 | なし | なし |
| 危険な操作のブロック | deny リストで禁止コマンドを指定可能 | なし | なし |

Claude Code は「どのファイルを読んでよいか」「どのコマンドを実行してよいか」を細かく制御できる。ChatGPT や Gemini にはこうした権限制御の仕組みがなく、誤って機密情報を入力してしまうリスクを設定レベルで防ぐことができない。

> **TIPS** — 「AI に社内データを読ませて大丈夫？」と聞かれたら、次の 3 点を伝えよう。(1) 入力データは学習に使われない、(2) ファイルは PC 上で処理されクラウドにアップロードされない、(3) 読み取り・実行の権限を細かく制御できる。

## Claude Code とは

![Claude Code ターミナルデモ](https://raw.githubusercontent.com/anthropics/claude-code/main/demo.gif)
*出典: [GitHub — anthropics/claude-code](https://github.com/anthropics/claude-code)*

### Claude の 4 つのインターフェース

Claude には使い方の異なる 4 つのインターフェースがある。本勉強会で扱う **Claude Code（CLI）** がなぜ最も強力なのかを理解するために、まず全体像を押さえる。

```
    自由度・できることの幅
    ▲
    │
    │  ★ Claude Code（CLI）
    │    AI が自律的に作業を遂行する
    │    ファイル操作・コマンド実行・Git 管理
    │
    │  ● Cowork（デスクトップ版の拡張機能）
    │    ファイル読み書き・マルチステップ実行
    │    コーディング以外のナレッジワーク向け
    │
    │  ■ デスクトップ版
    │    ショートカットで即呼び出し
    │    MCP 連携・Cowork 機能
    │
    │  ▼ Web 版
    │    ブラウザだけで利用可能
    │    チャット形式の質問応答
    │
    └──────────────────────────► 手軽さ
```

| 比較項目 | Web 版 | デスクトップ版 | Cowork | Claude Code（CLI） |
|---|---|---|---|---|
| 利用方法 | ブラウザで claude.ai にアクセス | 専用アプリをインストール | デスクトップ版内の機能 | ターミナルで `claude` コマンドを実行 |
| セットアップ | 不要 | アプリをインストール | デスクトップ版に含まれる | Node.js + CLI をインストール |
| 基本動作 | 1 問 1 答のチャット | チャット + ショートカット呼出 | 指示を出して放置 → 完成物を受取 | AI が自律的に複数ステップを遂行 |
| ファイル操作 | アップロード/ダウンロードのみ | アップロード/ダウンロードのみ | ローカルファイルの読み書き可能 | ローカルファイルの読み書き可能 |
| コマンド実行 | 不可 | 不可 | 限定的に可能 | あらゆるコマンドを実行可能 |
| Git 連携 | 不可 | 不可 | 不可 | コミット・ブランチ・PR 作成まで自動 |
| 外部ツール連携 | プラグイン | MCP 連携 | MCP + プラグイン | MCP 連携 |
| 並列実行 | 不可（タブは可） | 不可 | 不可 | 複数セッションを同時実行可能 |
| 向いている用途 | 質問・相談・文章添削 | 日常的な AI 活用・リサーチ | 資料作成・データ整理・リサーチ | 開発・自動化・業務フロー構築 |

#### 「AI モデル」と「AI エージェント」の違い

ニュースで目にする **Claude Opus** や **Claude Sonnet** は「AI モデル」— つまり頭脳にあたる部分である。一方 **Claude Code** は、その頭脳に**手足を与えた AI エージェント**。両者の関係を整理すると次のようになる。

| | AI モデル（Claude Opus / Sonnet 等） | AI エージェント（Claude Code） |
|---|---|---|
| 役割 | 質問に答える・文章を生成する「頭脳」 | 頭脳を使って実際に作業を遂行する「作業者」 |
| できること | テキストの入出力のみ | ファイル操作・コマンド実行・Git 管理まで |
| たとえ | 何でも知っている相談役 | 指示どおりに手を動かしてくれるアシスタント |

つまり、Claude Opus や Sonnet は「賢いけれど手が動かせない」存在で、Claude Code は「その賢さを使って実際に仕事をこなす」存在。本勉強会では後者の **Claude Code（AI エージェント）** を使って、実務を自動化するスキルを身につける。

### なぜチャット AI ではなくエージェント型 AI なのか

「質問に答える」チャット AI ではなく、**AI が作業者として動く**エージェント型ツール。ローカルのファイル構造・コマンド実行・Git 操作を含めた作業全体に統合される。

### チャット AI とエージェント型 AI の違い

| 比較項目 | 従来のチャット AI | Claude Code |
|---|---|---|
| 動作モデル | 質問に回答する | AI が作業者として動く |
| ファイル操作 | 不可 | ローカルファイルの読み書き可能 |
| コマンド実行 | 不可 | ターミナルコマンドを実行可能 |
| 開発ツール連携 | 不可 | Git, VS Code 等と統合 |
| 作業の自律性 | 1 問 1 答 | 複数ステップを自動で遂行 |

#### なぜ Claude Code（CLI）が最も強力なのか

1. **ファイルを直接操作できる** — アップロード・ダウンロードの手間なく、PC 上のファイルをそのまま読み書きする
2. **コマンドを実行できる** — Python スクリプトの実行、データ加工、ファイル変換など、ターミナルでできることはすべて任せられる
3. **Git でやり直しがきく** — すべての変更が履歴に残るため、失敗しても安全に元に戻せる
4. **複数の作業を同時に走らせられる** — 1 つのタスクを実行中に別のセッションで次の作業を開始できる
5. **スキルとして再利用できる** — 一度作ったワークフローを保存し、コマンド 1 つで何度でも再実行できる

#### Cowork と Claude Code の使い分け

Cowork は Claude Code と同じエージェント技術を使っているが、**ターミナル不要**で使える点が特徴。コーディングに馴染みがない人でも、デスクトップアプリ上で「この資料をまとめて」「このデータを整理して」と指示すれば、AI がファイルを直接操作して成果物を仕上げる。

ただし Cowork には**動作が遅い**という大きな欠点がある。Cowork はデスクトップアプリの UI を経由して AI とやり取りするため、画面の描画・ファイルの読み書き・結果の表示といった各ステップで UI のオーバーヘッドが発生する。一方 Claude Code（CLI）はターミナルで直接やり取りするため、UI のボトルネックがなく処理が速い。体感で **2〜3 倍の速度差**が出ることも珍しくない。

一方、Claude Code（CLI）は**制約が最も少なく、自由度が最も高い**。Git によるバージョン管理、複数セッションの並列実行、スキルによる再利用など、業務を本格的に自動化するための機能がすべて揃っている。

| 比較 | Cowork | Claude Code（CLI） |
|---|---|---|
| 操作方法 | GUI（デスクトップアプリ） | CUI（ターミナル） |
| 動作速度 | 遅い（UI 経由のオーバーヘッド） | 速い（直接実行） |
| 学習コスト | 低い | やや高い（ターミナル操作に慣れる必要あり） |
| Git 連携 | 不可 | コミット・ブランチ・PR まで自動 |
| 並列実行 | 不可 | 複数セッション同時実行可能 |

> 本勉強会では **Claude Code（CLI）** を使用する。ターミナル操作に慣れてしまえば、Cowork やデスクトップ版でできることはすべて Claude Code でもできる。逆はできない。速度面でも Claude Code が圧倒的に有利であり、実務で使い込むほどその差を実感する。

### エラーが出ても大丈夫

操作中に赤いエラーメッセージが表示されても、慌てる必要はない。Claude Code 自身がエラーの内容を理解できるため、次の 2 ステップで解決できる。

1. **エラーが出たら** → そのまま Claude Code に「どうすればいいですか？」と聞く
2. **解決策が提示されたら** → 「お願いします」と返す

これだけで Claude Code がエラーの原因を特定し、修正まで自動で行ってくれる。エラーの内容を自分で調べたり、意味を理解したりする必要はない。

> エラーメッセージの詳しい意味を知りたい場合は、[付録 C](#付録-cエラーメッセージ解説詳しく知りたい人向け)にまとめてある。ただし、まずは上の 2 ステップを試すだけで十分。

### 「手動の方が早い」と感じたときに思い出すこと

Claude Code に作業を依頼すると、1 つのタスクに約 10 分かかることがある。手動でやった方が早いと感じるかもしれないが、2 つの重要な利点がある。

**1. 再実行時の効率化**

初回は指示を細かく分けて試行錯誤するため時間がかかる。しかし一度うまくいったワークフローは、次回から一気に依頼できる。Claude Code ではこのワークフローを **スキル（Skills）** として保存でき、`/スキル名` で呼び出すだけで同じ作業を再現できる。

```
# 例：初回は試行錯誤しながら 30 分かけた LP 制作が…
/create-lp キャンペーン名「夏のセール」 商品数3 CTA「今すぐ申込」

# スキルとして保存しておけば、次回は 1 行の指示で完了する
```

手動作業は毎回同じ時間がかかるが、Claude Code は「やり方を覚えさせる」ことで回を重ねるほど速くなる。

**2. 並行作業が可能**

Claude Code が 10 分間タスクを実行している間、自分は別の作業ができる。ターミナルやデスクトップアプリで複数セッションを開けば、2 つ以上の作業を同時に走らせることも可能。

```
# セッション A：LP を作成中（10 分かかる）
ランディングページを作成してください…

# セッション B：その間に別の作業を開始
売上データの CSV を集計してグラフにしてください…
```

手動作業は 1 つずつしか進められないが、Claude Code なら **待ち時間がそのまま別の作業時間になる**。3 つの作業を並列で走らせれば、合計 30 分の作業が 10 分で終わる。

## 付録 B：ビジネスユーザー向け settings.json（コマンド許可設定）

Claude Code は変更操作の前に毎回許可を求める。頻繁に使う安全なコマンドを事前に許可しておくと、実務がスムーズに進む。

### B-1. 設定ファイルの場所

| スコープ | ファイルパス | 用途 |
|---|---|---|
| ユーザー設定（全プロジェクト共通） | `~/.claude/settings.json` | 個人の全プロジェクトに適用 |
| プロジェクト設定（チーム共有） | `プロジェクトルート/.claude/settings.json` | Git にコミットしてチームで共有 |
| ローカル設定（個人・非共有） | `プロジェクトルート/.claude/settings.local.json` | 自分だけの設定（gitignore 対象） |

> ビジネスユーザーの初心者は **ユーザー設定**（`~/.claude/settings.json`）に書くのが最もシンプル。

### B-2. 設定の適用方法

PowerShell で以下を実行してフォルダとファイルを作成する。

```powershell
# .claude フォルダがなければ作成
New-Item -ItemType Directory -Path "$env:USERPROFILE\.claude" -Force

# settings.json を作成（または上書き）
notepad "$env:USERPROFILE\.claude\settings.json"
```

メモ帳が開くので、以下のテンプレートを貼り付けて保存する。

もしくは、Claude Code のセッション内で `/permissions` コマンドを実行して UI から設定することもできる。

### B-3. おすすめ設定

`Bash(*)` ですべてのコマンドを一括許可し、deny でシステムを壊す致命的な操作だけをブロックするシンプルな構成。ファイル編集も自動承認されるため、確認ダイアログに煩わされずスムーズに作業できる。

```json
{
  "permissions": {
    "defaultMode": "acceptEdits",
    "allow": [
      "Bash(*)",
      "WebFetch"
    ],
    "deny": [
      "Bash(rm -rf /)",
      "Bash(rm -rf / *)",
      "Bash(rm -rf ~)",
      "Bash(rm -rf ~ *)",
      "Bash(rm -rf C:*)",
      "Bash(del /s /q C:\\*)",
      "Bash(sudo *)",
      "Bash(git push --force *)",
      "Bash(git push -f *)",
      "Bash(git reset --hard *)",
      "Bash(git clean -f *)",
      "Bash(git checkout . *)",
      "Bash(git restore . *)",
      "Bash(git branch -D *)",
      "Bash(format *)",
      "Bash(chmod 777 *)"
    ]
  },
  "language": "日本語"
}
```

**設計方針：allow で全許可 → deny で致命的操作だけブロック**

コマンドを一つ一つ allow に列挙する方式だと、許可漏れのたびに確認ダイアログが出て作業が止まる。`Bash(*)` で全許可してから deny で絞る方が実用的。

**allow の解説**

| ルール | 効果 |
|---|---|
| `Bash(*)` | すべての Bash コマンドを許可（deny リストで除外したものを除く） |
| `WebFetch` | すべてのドメインへの Web ページ取得を許可 |

**defaultMode の解説**

| 設定値 | 効果 |
|---|---|
| `acceptEdits` | ファイルの作成・編集・削除を確認なしで自動承認する |

> 一人で作業する場合にファイル変更前の確認を入れたいときは、この行を削除すればデフォルトモード（毎回確認）に戻る。

**deny の解説**

| ルール | ブロックする操作 | 理由 |
|---|---|---|
| `Bash(rm -rf /)`, `Bash(rm -rf / *)` | ルートディレクトリ `/` の全削除 | OS ごと消える致命的操作 |
| `Bash(rm -rf ~)`, `Bash(rm -rf ~ *)` | ホームディレクトリ `~` の全削除 | ユーザーデータが全消失する |
| `Bash(rm -rf C:*)` | Windows C ドライブの全削除 | 同上 |
| `Bash(del /s /q C:\\*)` | Windows del での C ドライブ全削除 | 同上 |

> `rm -rf /tmp/...` や `rm -rf ~/projects/mydir` など特定のパス配下の削除はブロックされない。ブロック対象は `/` と `~` そのものだけ。

| ルール | ブロックする操作 | 理由 |
|---|---|---|
| `Bash(sudo *)` | 管理者権限での実行 | システム設定の変更を防ぐ |
| `Bash(git push --force *)`, `Bash(git push -f *)` | 強制プッシュ | リモートの履歴を破壊する |
| `Bash(git reset --hard *)` | 強制リセット | 未コミットの変更を消失させる |
| `Bash(git clean -f *)` | 追跡外ファイルの強制削除 | 新規作成ファイルが消える |
| `Bash(git checkout . *)`, `Bash(git restore . *)` | 作業ツリーの全変更破棄 | 編集中の内容が消える |
| `Bash(git branch -D *)` | ブランチの強制削除 | マージ前のブランチが消える |
| `Bash(format *)` | ディスクフォーマット | ドライブ全体が消える |
| `Bash(chmod 777 *)` | パーミッションの全開放 | セキュリティリスク |

> 通常のファイル削除（`rm file.txt` や `rm -r mydir/`）はブロックされない。ブロックするのはルート・ホーム・ドライブ全体を消すパターンのみ。Git で履歴管理しているため、プロジェクト内の誤削除は復元可能。

### B-4. 設定の確認方法

Claude Code セッション内で `/permissions` を実行すると、適用されている許可・拒否ルールの一覧を確認できる。

```
/permissions
```

設定が正しく反映されているか、`/permissions` の出力で確認しておこう。

### B-5. パーミッション設定のルール記法まとめ

| 記法 | 意味 | 例 |
|---|---|---|
| `Bash(コマンド)` | 完全一致 | `Bash(git status)` |
| `Bash(コマンド *)` | 前方一致（引数あり） | `Bash(npm run *)` → `npm run build` 等にマッチ |
| `Bash(* サフィックス)` | 後方一致 | `Bash(* --version)` → 何でも `--version` にマッチ |
| `Read(パス)` | ファイル読み取り制御 | `Read(.env)` → .env の読み取りを制御 |
| `Edit(パス)` | ファイル編集制御 | `Edit(/src/**)` → src 配下の編集を制御 |

> deny は allow より優先される。同じコマンドが both に含まれる場合 deny が勝つ。

## 付録 C：エラーメッセージ解説（詳しく知りたい人向け）

基本的にはエラーが出たら Claude Code に「どうすればいいですか？」と聞くだけで解決する（→「エラーが出ても大丈夫」を参照）。ここでは、よく表示されるエラーメッセージの意味と対処法をまとめておく。

| エラーメッセージ | 意味 | 対処法 |
|---|---|---|
| `Author identity unknown` | Git に自分の名前・メールアドレスが登録されていない | `git config --global` で名前・メールを設定 |
| `'claude' is not recognized` | PC が Claude Code の場所を見つけられていない | PowerShell を再起動 or 再インストール |
| `fatal: not a git repository` | 今いるフォルダが Git で管理されていない | `git init` を実行 |
| `code: command not found` | PC が VS Code の場所を見つけられていない | VS Code を再インストール（PATH 追加にチェック） |

## 付録 D：Claude Code 基本コマンド

### 自動承認モード

`--enable-auto-mode` を付けて起動すると、すべての操作が確認なしで自動実行される。

| 起動方法 | ファイル編集 | コマンド実行 | 用途 |
|---|---|---|---|
| `claude --enable-auto-mode` | 自動承認 | 自動承認 | 実務・勉強会（推奨） |
| `claude` | 毎回確認 | 毎回確認 | 内容を逐一確認したいとき |

> 自動承認モードでも、[付録 B](#付録-bビジネスユーザー向け-settingsjsonコマンド許可設定) の settings.json で deny に設定した致命的操作（`rm -rf /`、`git push --force` 等）はブロックされる。

### Shift + Tab でモードを切り替える

Claude Code は起動した後でも、**Shift キーを押しながら Tab キー** を押すたびに動作モードが切り替わる。プロンプトの入力欄の下に現在のモードが表示されるので、用途に応じて使い分ける。

```
[ Shift + Tab を押すたびに次のモードへ切り替わる ]

   通常モード（毎回確認）
        │
        ▼  Shift + Tab
   自動承認モード（auto-accept edits）
        │
        ▼  Shift + Tab
   プランモード（plan mode）
        │
        ▼  Shift + Tab
   通常モード（最初に戻る）
```

| モード | 表示 | ファイル編集 | コマンド実行 | 用途 |
|---|---|---|---|---|
| 通常モード | （表示なし） | 毎回確認 | 毎回確認 | 1 つずつ内容を確認しながら進めたいとき |
| 自動承認モード | `auto-accept edits on` | 自動承認 | 毎回確認 | 編集作業をテンポよく進めたいとき |
| プランモード | `plan mode on` | **実行しない** | **実行しない** | まず計画だけ立てて、内容を確認してから実行したいとき |

**プランモードの使いどころ**

プランモードは「ファイルもコマンドも一切変更せず、何をするかの計画だけを立てる」モード。大きな作業を頼む前にこのモードで一度プランを出させ、内容に納得してから Shift + Tab で通常モードや自動承認モードに戻して実行すると、想定外の変更を防げる。

> **TIPS** — 「いきなり全部任せるのは不安」というときは、まずプランモードで計画を見せてもらうのがおすすめ。計画に問題がなければ「この計画で進めてください」と伝えるだけで、すぐに作業を始めてくれる。

### /init コマンドと CLAUDE.md

`/init` を実行すると、Claude がプロジェクトの内容を分析し、`CLAUDE.md` ファイルを自動生成する。`CLAUDE.md` は Claude Code が毎回セッション開始時に読み込む設定ファイルで、以下のような情報が含まれる。

- プロジェクトの概要
- 使用している言語・フレームワーク
- ビルド・テストコマンド
- コーディング規約

> 新しいプロジェクトを始めるたびに `/init` を実行しておくと、Claude Code の精度が上がる。生成された `CLAUDE.md` は手動で編集してカスタマイズできる。テンプレート例は[付録 A](#付録-aビジネスユーザー向け-claudemd-テンプレート)を参照。

### 主要コマンド一覧

操作中に「あのコマンドなんだっけ？」と思ったら、このページを参照。

| コマンド | 機能 | 備考 |
|---|---|---|
| `claude --enable-auto-mode` | 自動承認モードで対話開始 | 本勉強会ではこちらを使用 |
| `claude` | 対話モードを開始 | 操作のたびに確認が入る |
| `claude "タスク"` | 1 回限りのタスクを実行 | 対話せず結果だけ受け取りたいとき |
| `claude -p "質問"` | 質問に回答して終了 | 調べ物に便利 |
| `claude -c` | 直近の会話を続行 | 前回の続きをやりたいとき |
| `claude -r` | 過去のセッションを選択して再開 | 数日前の作業を再開したいとき |
| `claude commit` | Git コミットを作成 | メッセージも自動生成される |
| `/clear` | 会話履歴をクリア | 別の話題に切り替えるとき |
| `/help` | ヘルプを表示 | 困ったらまずこれ |
| `Esc` | 実行中のアクションを停止 | AI の動作を途中で止められる |
| `Esc + Esc` | 巻き戻しメニューを開く | 直前の操作をやり直したいとき |

### コンテキスト管理のコツ

- 別のタスクに移るときは `/clear` でリセットする
- 長い会話で精度が落ちたら `/compact` で要約する
- 前の会話を続けるなら `claude -c` で再開する

## 付録 E：効果的なプロンプト設計

Claude Code への指示の出し方を工夫するだけで、成果物の質が大きく変わる。

### プロンプトの基本原則

#### 具体的に指示する

| NG | OK |
|---|---|
| メールを作って | セミナー案内メールを作ってください。対象は中小企業の人事担当者、丁寧だが堅すぎないトーンで |
| ページを直して | seminar-lp.html の「申し込みボタン」の色を青から緑に変更してください |
| いい感じにして | proposal.html のフォントサイズを本文 16px、見出し 24px に統一してください |

#### 段階的に指示する

複雑なタスクは一度に全部頼まず、ステップに分けて指示する。

```
1. evaluation-data.csv を読み込んで、部署別の平均評価点を集計してください
2. 集計結果を表形式の HTML レポートにしてください
3. レポートに棒グラフを追加してください（Chart.js を使用）
```

#### 完成イメージを伝える

```
セミナー案内メールを作成してください。
- 件名に「無料」「人事評価」のキーワードを入れる
- 300 文字以内に収める
- 文末に申込 URL を配置する
作成後、seminar-mail.txt として保存してください
```

### 調べる→計画→実行のフロー

大きなタスクは「いきなり作って」ではなく、段階を踏むと精度が上がる。

1. **調べる**：まず Claude に現状を把握させる
2. **計画**：作業計画を作らせて内容を確認する
3. **実行**：計画に沿って作業させる
4. **記録**：完了したらコミットする

```
# 調べる
このフォルダにあるファイルの一覧と、それぞれの内容を教えてください

# 計画
これらのファイルを使って営業用の提案書を作りたい。どういう構成にするか計画を立ててください

# 実行
計画どおりに提案書を proposal.html として作成してください

# 記録
わかりやすいメッセージでコミットしてください
```

## 付録 F：サブスクリプション詳細

### なぜサブスクリプションが重要なのか

Claude Code の強さは「AI が自律的に考えながら何度もやり取りを繰り返す」ところにある。1 つのタスクで AI モデルへの問い合わせが数十回〜数百回発生することも珍しくない。無料プランの制限枠ではこの動作をまかなえず、途中で停止してしまう。

### プランごとの性能・利用量の違い

| | Free | Pro | Max | Team / Enterprise |
|---|---|---|---|---|
| 利用可能なモデル | 基本モデルのみ | 最新モデル | 最新モデル | 最新モデル |
| 利用量の上限 | 非常に少ない | 標準 | 5 倍〜20 倍 | チーム管理で柔軟に配分 |
| Claude Code | 不可 | 可 | 可（長時間タスク向き） | 可（管理者による一括設定） |

> **TIPS** — Claude の Web チャットは無料で試せるが、Claude Code（AI エージェント）は有料プラン専用。AI が自分で考えて何十回も作業を繰り返す仕組みなので、無料枠では動作を支えきれない。業務で本格的に使うなら、サブスクリプションは必要な投資と考えよう。
