# Claude Code 実践トレーニング（macOS + ターミナル版）

## セクション 1：macOS 環境セットアップ（40 分）

macOS には標準で **ターミナル.app** が入っており、Claude Code は基本的にこのターミナルから操作します。VS Code などのエディタは不要です。

### 1-1. ターミナルアプリを起動する

macOS の標準アプリ「**ターミナル.app**」を使います。

**起動方法（どちらでも OK）**

1. **Spotlight 検索**（推奨）
   - `Cmd + Space` で Spotlight を開く
   - 「ターミナル」と入力 → Enter

2. **Finder から起動**
   - Finder →「アプリケーション」→「ユーティリティ」→「ターミナル.app」をダブルクリック

> **Tips**: 起動したターミナル.app を **Dock のアイコンを右クリック → オプション → Dock に追加** しておくと、次回からワンクリックで開けます。

### 1-2. ターミナルの基本画面

ターミナルを開くと、以下のような画面が表示されます。

```
ユーザー名@MacBook-Pro ~ %
```

| 表記 | 意味 |
|---|---|
| `ユーザー名@MacBook-Pro` | ログインユーザー名と Mac の名前 |
| `~` | 今いるフォルダ（ホームディレクトリ＝`/Users/<ユーザー名>`） |
| `%` | コマンド入力待ちの記号（プロンプト） |

> macOS のターミナルは初期状態で **zsh**（ズィーシェル）というシェルが動いています。Claude Code は zsh / bash どちらでも動作します。

### 1-3. 覚えておきたい基本コマンド 4 つ

Claude Code を使う前に、ターミナルでよく使う 4 つのコマンドを押さえておきます。

| コマンド | 意味 | 使用例 |
|---|---|---|
| `pwd` | 今いるフォルダのフルパスを表示 | `pwd` → `/Users/taro` |
| `ls` | フォルダ内のファイル一覧を表示 | `ls` / `ls -la`（隠しファイルも表示） |
| `cd` | フォルダを移動する | `cd ~/Desktop` / `cd ..`（一つ上へ） |
| `mkdir` | フォルダを作成する | `mkdir my-folder` |

**試してみる**

```ターミナル
pwd
ls
cd ~/Desktop
pwd
```

> `~`（チルダ）は「自分のホームディレクトリ」のショートカット。`cd ~` だけで `/Users/<ユーザー名>` に戻れます。

### 1-4. プロジェクトフォルダの準備

研修用のフォルダを作成して、そこに移動します。

```ターミナル
mkdir -p ~/Desktop/claude-training
cd ~/Desktop/claude-training
pwd
```

`pwd` の表示が `/Users/<ユーザー名>/Desktop/claude-training` となっていれば成功です。

> **重要：Claude Code の操作範囲について**
>
> Claude Code は、起動したフォルダ（＝ `pwd` で表示される場所）配下のファイルを自由に読み書きします。フォルダ内のファイルは Claude Code によって削除・追加・変更される可能性があります。逆に、起動したフォルダの外にあるファイルやフォルダには許可なくアクセスすることはありません。そのため、**業務で使用する重要なファイルがあるフォルダで直接 Claude Code を起動せず、必ず専用のフォルダを作成して作業する**こと。

### 1-5. Claude Code のインストール（curl ネイティブインストーラ）

Anthropic が現在推奨しているインストール方法は、`curl` で公式の配布スクリプトを取得して実行する **ネイティブインストール** です。Node.js やパッケージマネージャ（Homebrew など）の事前インストールは不要で、コマンド 1 行でインストールが完了します。

```ターミナル
curl -fsSL https://claude.ai/install.sh | bash
```

> **このコマンドは何をしているのか？**
> - `curl -fsSL <URL>` … 指定 URL のスクリプトを取得して標準出力に流す
> - `| bash` … 取得したスクリプトを bash で実行する
> - 結果として、自分の Mac 用にビルドされた `claude` バイナリが `~/.local/bin/claude` に配置されます

インストール完了後、ターミナルを **一度閉じてから開き直し**、以下で確認します。

```ターミナル
claude --version
```

バージョン番号が表示されれば成功です。

**ネイティブインストールのメリット**

- **Node.js / npm が不要** — Mac に開発環境を構築しなくてよい
- **自動更新** — 起動時にバックグラウンドで最新版に更新される
- **公式が現在もっとも推している方法** — トラブルが少ない

> **`zsh: command not found: claude` と出る場合**: ターミナルを開き直しても解決しないときは、PATH が通っていない可能性があります。以下を実行してから再度ターミナルを開き直してください。
>
> ```ターミナル
> echo 'export PATH=$HOME/.local/bin:$PATH' >> ~/.zshrc
> source ~/.zshrc
> claude --version
> ```

> **より詳しい確認**: `claude doctor` を実行すると、インストール状態・設定・依存関係のチェック結果がまとめて表示されます。

### 1-6. 初回ログインと起動

研修用フォルダで Claude Code を起動します。

```ターミナル
cd ~/Desktop/claude-training
claude
```

初回起動時の流れは以下のとおり。

1. 認証方法の選択肢が表示される → **「Claude.ai Subscription」** を矢印キーで選んで Enter
2. **ブラウザが自動で開く** → Claude アカウント（会社の Google Workspace アカウント）でログイン
3. ブラウザで「承認する」ボタンを押す
4. ターミナルに戻ると、Claude Code のプロンプトが表示されている

ログインに成功したら、Claude Code 内で `/init` を実行します。

```
/init
```

> `/init` を実行すると、Claude が今のフォルダの内容を分析し、設定ファイル（`CLAUDE.md`）を自動生成します。

### 1-7. ターミナル操作で覚えておくキー

Claude Code を使うときに頻繁に使うキー操作です。

| キー | 用途 | 補足 |
|---|---|---|
| **Esc** | Claude の応答を途中で止める | 意図しない方向に進み始めたら押す |
| **Control + C** | 入力のキャンセル / Claude Code の終了 | 1 回でカーソル行をクリア、何もない状態で 2 回連続で Claude Code を終了 |
| **Cmd + K** | ターミナル画面のクリア | Claude Code の会話履歴は消えない（見た目だけ整理される） |
| **Cmd + T** | 新しいタブを開く | Claude Code を動かしたまま別タブで `git` 等を叩きたいとき |
| **Cmd + +/-** | 文字サイズの拡大・縮小 | 受講中、画面共有時の見やすさ調整に |

> ⚠️ **注意**: ターミナルでは Claude Code への割り込みは **Command + C ではなく Control + C** です。Command + C は OS 標準のコピー操作なので、間違えると Claude が止まりません。逆に「画面に出ている文字をコピーしたい」ときは、選択して Command + C で OK です。

## セクション 2：はじめての操作（15 分）

![Claude Code デモ](https://raw.githubusercontent.com/anthropics/claude-code/main/demo.gif)
*出典: [GitHub — anthropics/claude-code](https://github.com/anthropics/claude-code)*

ターミナルで `claude` を起動した状態で、以下の指示をそのまま入力していきます。

### 2-1. 競合を調べさせてみる

Claude Code に以下を入力します。AI がリアルタイムで Web サイトを読み込み、分析してくれます。

```
以下の3社のWebサイトを読んで、人事評価サービスの競合比較表を作ってください。
比較項目は「サービス概要」「主な機能」「ターゲット企業規模」「料金」とします。
markdownファイルとして保存してください。
- https://smarthr.jp/
- https://www.kaonavi.jp/
- https://hcm-jinjer.com/
```

> **markdown（マークダウン）ファイルとは？**
> テキストに簡単な記号を付けて見出しや箇条書きを表現する軽量な文書形式（拡張子 `.md`）。Word のように専用ソフトが不要で、テキストエディットでも編集できます。Claude Code との相性が良く、AI が生成・修正しやすいため、本勉強会ではドキュメント作成の基本形式として使います。

**許可を求められたら Yes を選択**

作成されたファイルが研修用フォルダに保存されます。確認方法は次のとおり。

```ターミナル
# 別タブで（Cmd + T）以下のコマンドを実行
cd ~/Desktop/claude-training
ls           # ファイル一覧を確認
open .       # 今いるフォルダを Finder で開く
```

> **Finder で見る方法**: `open .`（ドット）は「今いるフォルダを Finder で開く」コマンド。出来上がった markdown ファイルをダブルクリックすると、テキストエディットで内容を確認できます。

> **ポイント**: Claude Code は 3 社の Web サイトをリアルタイムに読み取って比較表にまとめてくれます。手作業なら数時間かかる競合調査が、1 回の指示で完成します。

### 2-2. 自動承認モードに切り替える

ここまでの操作で、Claude Code が何かを実行するたびに「Yes / No」の許可を求められたはずです。Web 検索やファイル作成のたびに毎回許可するのは手間がかかります。

**方法 1：起動時に自動承認モードで開始する（おすすめ）**

一度 Claude Code を抜けて（Control + C を 2 回）、以下のコマンドで起動し直します。

```ターミナル
claude --enable-auto-mode
```

これで、ファイル編集とコマンド実行が自動承認されます。

**方法 2：セッション中にモードを切り替える**

Claude Code 起動中に `Shift + Tab` を押すと、モードが順番に切り替わります。

| モード | 動作 | 用途 |
|---|---|---|
| Ask before edit | すべての操作で許可を求める | 慎重に確認したいとき |
| **Edit automatically** | ファイル編集は自動承認、コマンド実行は許可を求める | **本勉強会ではこちらを推奨** |
| Plan mode | 実行前に計画を提示し、承認後に実行 | 大きなタスクで事前に内容を確認したいとき |

> 以降の演習は **Edit automatically** モードで進めます。操作のたびに止まらないので、テンポよく作業できます。

### 2-3. 競合比較表を PDF にする

```
先ほど作った競合比較表の markdown ファイルを PDF に変換してください
```

Claude Code が変換に必要なツール（`pandoc` など）の導入から面倒を見てくれます。途中でコマンド実行（インストーラの実行など）の許可を求められたら、内容を確認したうえで Yes を選択してください。

### 2-4. PDF をプレビューで確認する

macOS には **プレビュー.app** という標準アプリがあり、`open` コマンドで PDF を開けます。

```ターミナル
# Claude Code セッションを抜けるか、別タブで実行
ls *.pdf                                  # PDFファイル名を確認
open 競合比較表.pdf                       # ファイル名は実際に作られたものに置き換え
```

または **Finder でファイルを 1 回クリックしてスペースキー** を押すと、**QuickLook（クイックルック）** で PDF をプレビューできます。これは macOS 標準の機能で、アプリを開かずに中身を確認できる便利な仕組みです。

### 2-5. ドキュメント作成の基本フロー

PDF や Word ファイルを直接編集するのではなく、**markdown ファイルを編集してから変換する** のが効率的なワークフローです。

1. markdown ファイルを作成（Claude Code に指示）
2. markdown から PDF・Docs・Excel などに変換（Claude Code に指示）
3. 変換後のファイルを確認しながら markdown ファイルを修正
4. 修正した markdown ファイルを再変換

> **Markdown のプレビュー方法（macOS 標準）**: Finder で `.md` ファイルを 1 回クリック → スペースキーで QuickLook が開きます。Markdown は記号付きのテキストとしてそのまま表示されますが、見出しや箇条書きの構造はそのまま読めるレベルです。

> **より高機能なプレビューが欲しいとき**: `Marked 2`（有料 / App Store）や `Typora` などの Markdown 専用エディタを使うと、編集とプレビューを同じ画面で行えます。本勉強会では標準機能だけで十分です。

## 付録 A：インストール状態の確認とトラブルシューティング

本編 1-5 の curl ネイティブインストールが完了しているかを確認するためのコマンドと、よくあるハマりどころです。

### A-1. インストール状態の確認

```ターミナル
claude --version       # バージョン番号が出るか
claude doctor          # 詳細な診断（推奨）
which claude           # バイナリの場所（通常は /Users/<ユーザー名>/.local/bin/claude）
```

`claude doctor` はインストール先・PATH・依存関係・自動更新の状態をまとめて確認できる公式の診断コマンドです。トラブル時はまずこれを実行して結果を見るのが近道です。

### A-2. PATH が通らないときの対処

`claude --version` で `command not found` が出る場合の手順です。

```ターミナル
# 1. バイナリ自体は配置されているか確認
ls -la ~/.local/bin/claude

# 2. PATH に ~/.local/bin が含まれているか確認
echo $PATH | tr ':' '\n' | grep -F .local/bin

# 3. 含まれていなければ ~/.zshrc に追加
echo 'export PATH=$HOME/.local/bin:$PATH' >> ~/.zshrc
source ~/.zshrc
claude --version
```

### A-3. インストールし直したいとき

curl ネイティブインストーラはそのまま再実行すれば最新版に上書きされます。

```ターミナル
curl -fsSL https://claude.ai/install.sh | bash
```

完全にアンインストールしたい場合は以下。

```ターミナル
rm -f ~/.local/bin/claude
rm -rf ~/.local/share/claude
```

> 設定ファイル（`~/.claude/` や `~/.claude.json`）は残るので、設定もまとめて消したいときは追加で `rm -rf ~/.claude ~/.claude.json` を実行します。ただしスキルや MCP 設定もすべて消えるので注意。

## 付録 B：知っておくと便利な macOS 標準コマンド

Claude Code と組み合わせて使うと役立つ macOS 標準コマンドです。

| コマンド | 用途 | 例 |
|---|---|---|
| `open <パス>` | ファイル / フォルダを Finder や関連アプリで開く | `open .` / `open report.pdf` |
| `pbcopy` | 標準入力をクリップボードにコピー | `cat memo.md \| pbcopy` |
| `pbpaste` | クリップボードの内容を出力 | `pbpaste > memo.txt` |
| `say "テキスト"` | 文字を音声で読み上げる | `say "完了しました"` |
| `caffeinate` | スリープを抑制 | 長時間処理中に `caffeinate -d` |

> Claude Code の処理が長くなりそうなときは `caffeinate -d` を別タブで実行しておくと、Mac がスリープに入って処理が止まる事故を防げます。
