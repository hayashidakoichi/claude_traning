# Claude Code 実践トレーニング

## セクション 1：Windows 環境セットアップ（40 分）

### 1-1. Git for Windows のインストール

1. https://git-scm.com/downloads/win にアクセス
2. インストーラをダウンロードして実行（デフォルト設定で OK）
3. PowerShell を開いて確認：

```powershell
git --version
```

4. ユーザー情報を設定：

```powershell
git config --global user.name "自分の名前"
git config --global user.email "自分のメールアドレス"
```

### 1-2. Claude Code のインストール

**方法 A：PowerShell（推奨）**

```powershell
irm https://claude.ai/install.ps1 | iex
```

**方法 B：コマンドプロンプト**

```batch
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd
```

**方法 C：WinGet**

```powershell
winget install Anthropic.ClaudeCode
```

### 1-3. 初回ログイン

```powershell
claude
```

初回起動時にブラウザが開く。Claude アカウント（会社のGoogle Workspaceアカウント）でログイン後、ターミナルに戻ると利用開始できる。

### 1-4. VS Code のインストールと連携

1. https://code.visualstudio.com/ からインストール
2. VS Code を起動 → `Ctrl+Shift+X` で拡張機能を開く
3. 「Claude Code」で検索してインストール
4. `Ctrl+Shift+P` → 「Claude Code: 新しいタブで開く」を選択

### 1-5. エクスプローラーでプロジェクトフォルダの準備

C:\Users\自分のユーザー名 のフォルダで、右クリックして「新しいフォルダ」を選択して「claude_training」という名前のフォルダを作成

### 1-6. VS Code で Claude Code を起動する

1. VS Code を起動し、「ファイル」→「フォルダーを開く」から `claude_training` フォルダを選択
2. メニューからターミナルを開く
3. ターミナルで以下を順に実行：

``` cmd
git init
claude --enable-auto-mode
```

4. Claude Code が起動したら `/init` を実行する：

```
/init
```

> `--enable-auto-mode` や `/init` の詳細は [Claude Codeの説明.md](Claude%20Codeの説明.md) を参照。

## セクション 4：データ整理・集計（30 分）

### 4-1. サンプルデータの作成

```
架空の人事評価データを evaluation-data.csv として作成してください：
- 30 名分のデータ
- 列：社員番号, 氏名, 部署（営業/開発/管理/企画）, 等級（S/A/B/C/D）, 上期評価点（1〜5）, 下期評価点（1〜5）, コメント
- 日本人の名前を使用
```

### 4-2. データの集計

```
evaluation-data.csv を読み込んで、以下の集計を行いターミナルに表示してください：
- 部署別の平均評価点（上期・下期それぞれ）
- 等級別の人数分布
- 上期から下期で評価点が上がった社員の一覧
```

### 4-3. 集計結果のレポート化

```
先ほどの集計結果を、上司に報告するための HTML レポートにしてください：
- タイトル「人事評価サマリーレポート」
- 部署別の平均評価点を表形式で表示
- 等級分布を一覧表示
- 印刷してもきれいに見えるデザイン
```

### 4-4. データの加工・変換

```
evaluation-data.csv から、等級が C または D の社員だけを抽出して
follow-up-list.csv として出力してください。
あわせて、各社員に対する「フォローアップ面談のご案内」メールのテンプレートを
follow-up-mail.txt として作成してください
```

### おすすめ課題：「こんなことまでできるの！？」体験

以下は、Claude Code の応用力を体感できる課題。
普通ならエンジニアや外注に依頼するレベルの成果物が、自然言語の指示だけで作れることを実感してほしい。

#### 課題例 E：CSV からインタラクティブダッシュボードを一発生成

セクション 4 で作った `evaluation-data.csv` を使い、クリックやフィルタ操作ができるリッチな HTML ダッシュボードを作る。

```
evaluation-data.csv を読み込んで、以下の要件でインタラクティブなダッシュボードを dashboard.html として作成してください：
- 部署別の平均評価点を棒グラフで表示（Chart.js を CDN で読み込み）
- 等級別の人数分布を円グラフで表示
- 部署をクリックすると、その部署の社員一覧がフィルタ表示される
- 上期→下期の評価推移を折れ線グラフで表示
- 全体の統計サマリー（平均点、最高点、最低点）をカード形式で上部に表示
- ダークモード切り替えボタン付き
- プロフェッショナルなデザイン
```

> **驚きポイント**: 静的な CSV が、クリック操作できるグラフ付きダッシュボードに変わる。普通なら BI ツール（Tableau 等）やエンジニアに依頼する作業。

#### 課題例 F：30 名分の個別フィードバックレターを一括自動生成

```
evaluation-data.csv の全30名分について、以下のルールで個別フィードバックレターを一括生成してください：
- 1人1ファイル（feedback-letters/社員番号_氏名.html）
- S/A評価の社員：成果を称え、次期の期待を伝えるポジティブなトーン
- B評価の社員：安定した貢献を認めつつ、成長領域を提案するトーン
- C/D評価の社員：改善点を具体的に提示し、サポート体制を伝える丁寧なトーン
- 各レターに社員名・部署・等級・評価点を差し込み
- 上期→下期で評価が上がった社員にはその点を特記
- 印刷可能なA4レイアウト
- フィードバックレター一覧ページ（index.html）も作成
```

> **驚きポイント**: 30 人分の個別文書が一括で、しかも評価に応じてトーンまで変えて生成される。手作業なら丸 1 日かかる作業。

#### 課題例 G：競合サイト分析レポートを自動作成

```
以下の3社のWebサイトを読み込んで、Scale人事評価の競合分析レポートを competitor-report.html として作成してください：
- https://hcm-jinjer.com/
- https://smarthr.jp/
- https://www.kaonavi.jp/
比較項目：サービス概要、主な機能、ターゲット企業規模、料金体系、導入実績、差別化ポイント
分析結果を比較表にまとめ、Scale人事評価のポジショニング戦略提案も含めてください
```

> **驚きポイント**: 実在の Web サイトからリアルタイムに情報を取得して分析レポートを自動生成。コンサルや調査会社に依頼するレベルの成果物。

#### 課題例 H：研修の成果物をまとめて PowerPoint 営業資料に変換

研修中に作成したファイル（提案書、集計レポート、競合分析など）を読み込ませて、コンサルファーム風のプロフェッショナルな PowerPoint 営業資料にまとめ上げる。

```
/mckinsey-slides このフォルダ内のファイルをすべて読み込んで、Scale人事評価の営業用プレゼンテーションを presentation.pptx として作成してください：
- スライド1：表紙「人事評価制度DX — Scale人事評価のご紹介」
- スライド2：課題提起（proposal.html の「課題」セクションを元に）
- スライド3：市場データ（competitor-report.html の競合情報を元に）
- スライド4：ソリューション概要（proposal.html の「解決策」を元に）
- スライド5：主要機能の紹介（proposal.html の「サービス概要」を元に）
- スライド6：競合比較（competitor-report.html の比較表を元に）
- スライド7：人事評価データの分析結果（evaluation-data.csv の集計結果を元にグラフで）
- スライド8：導入ステップ（proposal.html の「導入ステップ」を元に）
- スライド9：料金（要相談と記載）
- スライド10：お問い合わせ・次のステップ
- Colorfulbox のコーポレートカラーを使用
- 各スライドにはスライドノート（発表者用メモ）も記載
```

> `/mckinsey-slides` は McKinsey/BCG のようなコンサルティングファーム風の高品質スライドを生成するスキル。データドリブンなレイアウトと戦略フレームワークを自動適用する。

> **驚きポイント**: 研修中にバラバラに作った成果物が、コンサルファーム品質の営業用 PowerPoint に統合される。「今日半日で作ったものが、そのまま営業資料になる」という体験。ファイルを横断的に読み込んで再構成する Claude Code の真骨頂。
